Overview

The LONPFC Window Programs are specialized user interface components that provide interactive functionality for loan processing operations. These window-based programs offer focused, context-sensitive interfaces for specific loan processing tasks.

Program Categories

Category Programs Purpose
Documentation MEMOSC Memo entry and management for loan documentation
Collateral TITLEW Title processing for payoffs and collateral management
Escrow ESAPPL Escrow application and management interface
Information MAKERS Borrower and co-maker information display

Common Design Principles

  • Window-Based Interface: Modal windows with focused functionality
  • Context-Sensitive: Tailored to specific loan processing contexts
  • User-Friendly: Intuitive navigation and data entry
  • Data Validation: Real-time validation and error handling
  • Integration: Seamless integration with main loan processing flow

Usage Context: These window programs are typically called from LONPFC during payment processing to handle specialized tasks that require user interaction and data entry.

MEMOSC - Memo Entry Window

Program Details

Property Value
Program ID MEMOSC
Location WI/MEMOSC.CBL
Purpose Memo Inquiry / Input
Date Written 122391

Functionality

MEMOSC provides comprehensive memo management capabilities for loan accounts, allowing users to enter, view, edit, and delete various types of memos associated with loan processing activities.

Memo Types Supported

Memo Type Code Description Special Features
General Memos Standard General purpose loan memos 60 character limit (51 for Regency)
Collection Memos Collection Collection-specific memos No date/sequence stopping
Notes Memos Notes Extended notes with 2000 character dates Enhanced date handling
Investor Memos Investor Investor-related memos 2000 character date support
Dealer Memos Dealer Dealer-specific memos Toggle based on GP flag
Reversal Memos R Transaction reversal memos Auto-generated change memos
Collateral Memos T Collateral-related memos Collateral tracking

Key Features

  • Password Protection: Deletion of backdated memos requires password verification
  • Automatic Generation: Can auto-generate change memos for reversals
  • User ID Integration: Includes user ID in memos (Regency implementation)
  • Phone Format Display: Contact phone numbers displayed in proper format
  • Sequence Management: Handles memo sequencing including sequence 99 handling

Special Implementations

  • Regency: Uses smaller 51-character memos to include user ID
  • First Metro: Special form WI/MEMOFM with FMETRO-MEMO field
  • Modern: Court number validation with LIBSP/COURTW
  • PCCREDIT: Collateral memo support

TITLEW - Title Processing Window

Program Details

Property Value
Program ID TITLEW
Location WI/TITLEW.CBL
Purpose Payoff Title Processing (F9-Collateral, Page 3)
Window Type Modal Window

Functionality

TITLEW handles title processing for loan payoffs, managing collateral title information, lien release processing, and title forwarding instructions.

Core Functions

Function Description Usage Context
Title Information Entry Capture title details for payoff processing Loan payoff scenarios
Lien Release Processing Manage lien release documentation Collateral release
Title Forwarding Set up title forwarding instructions Customer or lienholder delivery
Corporate Printing Arrange corporate title printing Regency implementation

Data Fields Managed

  • LT-L-SEND-ASAP: Expedited title processing flag
  • LT-L-TITLE-SEND-FMNO: Title forwarding form number
  • LT-L-TITLE-ADR2: Additional address information
  • LT-L-RELEASE-TYPE: Type of lien release
  • LT-L-CARRIER-TYPE: Shipping carrier information
  • LT-L-CARRIER-ACCTNO: Carrier account number

Special Processing

  • Posting Mode: In posting mode (PO/IN), updates lienholder information
  • Non-Posting Mode: Modifies display without updating send-to information
  • Address Validation: Forces entry of complete address information
  • Memo Generation: Automatically creates memos with title processing details

ESAPPL - Escrow Application Window

Program Details

Property Value
Program ID ESAPPL
Location LIBWI/ESAPPL_*.CPY (Screen definitions)
Purpose Escrow Application Window
Components Screen, Evaluation, Definition, Working Storage

Functionality

ESAPPL provides an interface for managing escrow applications and escrow account processing within the loan management system.

Core Capabilities

  • Escrow Account Setup: Initial escrow account configuration
  • Payment Allocation: Allocation of payments to escrow accounts
  • Escrow Analysis: Periodic escrow account analysis
  • Shortage/Surplus Handling: Management of escrow imbalances
  • Disbursement Processing: Escrow fund disbursements

Copy Member Components

Component File Purpose
Screen Definition ESAPPL_SCN.CPY Window layout and field definitions
Evaluation Logic ESAPPL_EVA.CPY Field validation and processing logic
Data Definitions ESAPPL_DEF.CPY Data structure definitions
Working Storage ESAPPL_WKS.CPY Working storage variables

Integration Points

  • Payment Processing: Integrates with payment allocation routines
  • Account Management: Links to customer account information
  • Reporting: Supports escrow reporting requirements
  • Regulatory Compliance: Ensures RESPA compliance for escrow handling

MAKERS - Borrower Information Display

Program Details

Property Value
Program ID MAKERS
Location WI/MAKERS.CBL
Purpose Window Makers Display
Control Ctrl+N from Inquiry and Payment Posting

Functionality

MAKERS provides a comprehensive display window for borrower and co-maker information, including contact details, collateral information, and phone numbers.

Information Displayed

Information Type Source Details
Borrower Information BW Records Primary borrower details and contact information
Co-Maker Information BW Records Co-signer and guarantor information
Contact Information BW Records Phone numbers and contact preferences
Collateral Information LTC Records Vehicle and household goods collateral

Special Features

  • Collateral Display: Shows both household goods and vehicle collateral
  • Multiple Vehicles: Displays information for multiple cars
  • Phone Integration: Ctrl+N access to phone numbers window
  • Contact Management: Links to contact phone management
  • Record 3 Display: Shows collateral from record 3

Navigation Features

  • Ctrl+N: Opens phone numbers window (WI/PHONES or WI/PHONE2)
  • Contact Detection: Automatically determines appropriate phone window
  • Borrower Contact Phones: Displays contact phones from BW-REC
  • Multiple Contact Options: Supports various contact methods
