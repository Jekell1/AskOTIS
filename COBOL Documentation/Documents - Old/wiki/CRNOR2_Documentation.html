Program Overview

CRNOR2 is a utility program that builds NOFILE records and sets up payoff worker areas (POFF-WORKERS) for loan processing operations. This program is commonly called by other programs that need comprehensive loan information and payoff calculations without actually performing any updates.

Key Responsibilities

  • NOFILE Record Building: Constructs comprehensive loan information records
  • Payoff Worker Setup: Initializes and populates payoff calculation areas
  • Account Validation: Validates account accessibility and status
  • Path Configuration: Sets up necessary file paths for processing
  • Credit Restriction Processing: Handles available credit restrictions
  • Delinquency Analysis: Calculates delinquency and arrearage amounts
  • Branch Processing: Handles cross-branch account processing

Important: CRNOR2 is a setup utility that prepares data structures for other programs. It does not perform actual loan updates but provides comprehensive loan analysis and payoff preparation.

How to Use

CRNOR2 is called as a subroutine with specific setup requirements:

Standard Calling Sequence

MOVE  ACCOUNT NO     TO CRNOR2-ACCTNO
MOVE  SYSTEM DATE    TO CRNOR2-SYSDATE
SET   PAYOFF CODE    TO CRNOR2-LPTRCD          ----> 'PY' BY DEFAULT
SET     " " OR "Y"   TO CRNOR2-GET-PAYOFF-ONLY ----> ' '  BY DEFAULT
SET     " " OR "Y"   TO CRNOR2-RECAST-FG       ----> ' '  BY DEFAULT
SET     " " OR "Y"   TO CRNOR2-DEF-STOP   ----> ' '  BY DEFAULT
SET     " " OR "N"   TO CRNOR2-LPREC-FG 'N'  NO READ OF LPFILE
MOVE "CL/CRNOR2" TO FORM-NAM
CALL FORM-PROGX USING FORM-PATH CRNOR2-LINK NO-REC POFF-WORKERS
                                LN-REC.
CANCEL FORM-PROGX

POFF-WORKERS Output

The POFF-WORKERS structure is populated with payoff calculation data including:

  • Payoff Amounts: Principal, interest, and fee calculations
  • Rebate Information: Insurance and service charge rebates
  • Date Calculations: Payoff through dates and aging information
  • Account Status: Current loan status and restrictions

Processing Logic

Main Processing Flow

  1. Input Validation
    • Validate account number format
    • Check system date validity
    • Verify control flags
  2. Account Setup
    • Read loan record (LN-REC)
    • Validate account accessibility
    • Check for rescission loans (skip if found)
    • Set up cross-branch paths if needed
  3. File Path Configuration
    • Set LT-PATH and LP-PATH for cross-branch processing
    • Configure file access parameters
    • Handle global LP file settings
  4. NOFILE Record Building
    • Initialize NO-REC structure
    • Populate loan information
    • Calculate delinquency and arrearage
    • Set status and restriction flags
  5. Payoff Worker Setup
    • Initialize POFF-WORKERS structure
    • Calculate payoff amounts
    • Set up rebate calculations
    • Configure aging and delinquency data
  6. Special Processing
    • Handle recast processing if flagged
    • Process deferment policies
    • Apply available credit restrictions
    • Calculate industry penalties

Special Processing Features

  • Rescission Handling: Automatically skips rescission loans
  • Cross-Branch Support: Handles accounts across different branches
  • Available Credit Restrictions: Processes BW1 file for credit limits
  • Payoff Good Through: Special handling for payoff validity periods
  • First Payment Date Logic: Handles accounts without first payment dates
  • Global LP File: Supports global LP file processing

First Payment Date Issue: When calling from LNMAN2 without a first payment date, the program can enter an infinite loop. The program now sets to ORIG-1STPYDATE if it exists, or exits with an initialized NO-REC (S35Q-65).
